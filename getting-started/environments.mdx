# Environments

The Polymarket Exchange API is available in three environments. Use the appropriate endpoints based on your integration stage.

## Environment Summary

| Environment | Purpose | Auth0 Domain |
|-------------|---------|--------------|
| **Dev** | Development and testing | `pmx-dev01.us.auth0.com` |
| **Preprod** | Pre-production validation | `pmx-preprod.us.auth0.com` |
| **Prod** | Production trading | `pmx-prod.us.auth0.com` |

## Health Check Endpoint

All environments expose the same health check endpoint:

```
GET /v1/health
```

### Example Request

```bash
curl https://api.dev01.polymarketexchange.com/v1/health
```

### Example Response

```json
{
  "status": "SERVING",
  "version": "1.0.0"
}
```

## Quick Reference

### REST API Base URLs

| Environment | Base URL |
|-------------|----------|
| Dev | `https://api.dev01.polymarketexchange.com` |
| Preprod | `https://traderapi.us-east-1.privatelink.preprod.polymarketexchange.com` |
| Prod | `https://api.prod.polymarketexchange.com` |

### Auth0 Token Endpoints

| Environment | Token URL |
|-------------|-----------|
| Dev | `https://pmx-dev01.us.auth0.com/oauth/token` |
| Preprod | `https://pmx-preprod.us.auth0.com/oauth/token` |
| Prod | `https://pmx-prod.us.auth0.com/oauth/token` |

<Warning>
**Tokens must be refreshed every 3 minutes** across all environments.
</Warning>

## Environment Progression

We recommend the following integration progression:

1. **Dev** - Initial development and unit testing
2. **Preprod** - Integration testing and pre-launch validation
3. **Prod** - Live trading

Contact [support@qcex.com](mailto:support@qcex.com) to request access credentials for each environment.
